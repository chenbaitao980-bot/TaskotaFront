# 提案：AI 计划时间线改造与 WBS 思维导图

## 1. 用户问题 / 业务目标

### 问题

1. **时间约束解析错误**：用户要求"只在早上踢球并且只在这周踢球"，但 AI 生成的训练计划时间排到了 6 月 6 日（下下周）。本周日实际是 5 月 31 日，`_defaultPlanStart` 的 `today + index` 计算会导致超出本周范围，AI Prompt 中也缺少"本周"边界约束。
2. **"思维导图"命名和 UI 与用途不符**：当前 AI 聊天界面中展示计划流程图的区域被标记为"思维导图"，但它实际是按时间顺序排列的流程图（更像时间线），且时间顺序有时错乱，UI 尺寸较大。
3. **缺少真正的任务分解视图**：训练计划只有扁平化的表格和流程图，缺少体现"大目标→子任务→子动作"层次关系的 WBS 任务分解思维导图。

### 业务目标

- 让 AI 生成的学习/训练计划严格遵守用户的相对时间约束（这周、今天、早上等）
- 提供直观的时间线视图，按正确的时间顺序展示计划
- 提供 WBS 思维导图，帮助用户理解任务的层次结构和拆解思路

## 2. 目标用户、使用场景、触发入口

| 维度 | 说明 |
|---|---|
| **目标用户** | 在 AI 助手中请求生成学习/训练计划的用户 |
| **使用场景** | AI 生成一周训练计划后，用户查看和交互计划详情 |
| **触发入口** | AI 聊天界面中，当 AI 回复包含计划表格 (`[TABLE_BEGIN]...[TABLE_END]`) 时，自动渲染计划视图 |
| **交互流程** | AI 生成计划 → 展示计划表格（可编辑）→ 展示时间线 → 展示 WBS 思维导图 → 用户可一键分配任务 |

## 3. Scope

### 做什么

1. **修复时间约束解析**
   - 修改 `_defaultPlanStart`：当用户显式指定"这周"/"本周"时，计划日期不允许超出本周日（基于今天推算）
   - 修复 `_weekdayIndex`：将星期索引锚定到当前周的对应日期，而非从今天开始偏移
   - 更新 AI Prompt 指令，要求 AI 在生成日期时严格遵守用户指定的时间范围（这周、今天、早上等）
   - 增加 `6月6日` → 解析到下周的边界校验

2. **流程图 → 时间线改造**
   - 将"思维导图"标签改为"时间线"
   - 重新设计 UI：改为纵向时间轴样式，节点按时间顺序排列，视觉上更紧凑
   - 修复时间顺序排序逻辑，确保节点按 `row.start` 升序排列

3. **新增 WBS 任务分解思维导图**
   - 在计划视图底部新增"任务分解"区域
   - 以训练计划为输入，自动提取阶段/主题/内容，构建树形 WBS 结构
   - 用辐射状或树状节点图展示：**训练目标 → 阶段 → 训练项 → 子动作**

4. **计划表时间合并到同一行**
   - 当前计划表中"开始时间"和"结束时间"分两行展示（分开的标签/卡片）
   - 改为合并到同一行，格式如：`05-25 07:00 ~ 07:30` 或 `05-25 07:00 - 05-25 07:30`

### 不做什么

- 不改动日历页面（calendar_page）的时间线功能
- 不改动一键分配任务的标题生成逻辑（已有 `plan-table-time-edit-flow-fix` 修复）
- 不处理计划数据持久化（已有方案）
- 不修改 AI 的核心模型能力，只优化 Prompt 指令

## 4. 验收标准

| # | 标准 | 验证方式 |
|---|---|---|
| 1 | 用户说"只在这周"时，所有计划日期 ≤ 本周日（5/31） | 手动测试：输入"只在早上踢球并且只在这周踢球"，确认无任务超出 5/31 |
| 2 | "思维导图"标签已改为"时间线" | 肉眼检查 AI 聊天界面 |
| 3 | 时间线按日期/时间升序排列 | 手动验证：有明确日期时按日期排列，无明确日期时按 row 顺序 |
| 4 | 时间线 UI 更紧凑、纵向排列 | 肉眼检查：宽度缩窄，单列纵向布局 |
| 5 | WBS 思维导图显示层次化任务结构 | 手动验证：展开后可见阶段→项目→子项的树 |
| 6 | fluter analyze --no-fatal-infos 通过 | 自动检测 |
| 7 | flutter test 通过 | 自动检测 |

## 5. 关键取舍

| 取舍 | 选择 | 理由 |
|---|---|---|
| 时间线样式 | 纵向时间轴（非横向流程图） | 更符合"时间线"的视觉隐喻，移动端友好 |
| WBS 图实现方式 | 自定义 Widget 树状布局（非第三方包） | 避免额外依赖，保持与现有 UI 风格一致 |
| Prompt 约束 | 直接修改 system prompt 指令段（非动态注入） | 所有计划生成场景都需要此约束 |
| 时间约束优先级 | 用户显式指定的范围 > AI 推测 | 避免 AI 超出边界 |

## 6. 边界条件

- 计划行中没有显式日期时，使用 `_defaultPlanStart` 但截断到本周日
- 计划行超过本周容纳量时，截取到本周最后一天，超出的部分提示用户
- 用户未指定时间范围时，保持现有行为（从今天开始逐日递增）
- WBS 深度最多支持 4 层：目标 → 阶段 → 项目 → 子动作

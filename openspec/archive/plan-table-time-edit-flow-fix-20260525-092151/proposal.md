# plan-table-time-edit-flow-fix

## 为什么做
用户验证最新包后，计划生成和一键分配仍有 4 个问题：
1. 计划表只展示日期，没有展示具体开始/结束时间，需要精确到分钟。
2. 思维导图或流程图没有展示完整计划节点，目前只展示到周日，后续节点被截断。
3. 一键分配任务时把“主题”也带进了任务标题，导致第一个任务标题变成“主题”，这是错误的。
4. 计划表中的时间、主题、训练内容需要可以直接修改调整，再用于后续任务分配。

## 影响范围
- `lib/presentation/pages/ai_chat/ai_chat_page.dart`：计划表解析、计划表展示、流程图展示、一键分配预览和任务草稿生成。
- `lib/services/ai_service.dart`：可能需要补强 AI 输出契约，要求输出可解析的开始/结束时间到分钟。
- 回归用例：新增本次四个问题的验证步骤。

## 业务关系
- 主规格：`openspec/specs/smart-butler/spec.md`
- 命中能力：AI 任务拆解
- 变更类型：修改现有行为，不改登录、日历存储、任务模型数据库结构。

## 验收标准
- 计划表每行展示开始时间和结束时间，时间精确到分钟，例如 `05-25 09:00` 到 `05-25 10:00`。
- 流程图或思维导图展示 AI 返回的全部计划节点，不因为默认截取 7 条或周日而丢失后续内容。
- 一键分配任务时，任务标题只使用真正的训练主题或动作标题，不使用表头“主题”。
- 计划表中的时间、主题、训练内容可以在界面中直接编辑，编辑后的内容用于一键分配任务。
- `flutter analyze --no-fatal-infos` 通过。
- `flutter test` 通过或明确说明阻塞原因。
- GitNexus 变更检测结果不超出预期范围。

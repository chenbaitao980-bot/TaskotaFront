# plan-card-scrollable-timeline 设计方案

## 概述

在 AI 计划卡片内，将时间线模块和计划表模块改为超出有限高度后具备滚动/分页能力，确保内容不被遮挡。

## 时间线模块（`_buildTimelineView`）改造

### 现状
```dart
SingleChildScrollView(scrollDirection: Axis.horizontal, child: Row(children: [...]))
```
水平滚动，内容超出可视区域时用户需要手动横向滑动才发现后续内容。

### 目标
固定高度内垂直滚动展示，让时间线卡片在卡片内容区域内自然"弯弯绕绕"地向下排列。

### 设计
- 移除 `SingleChildScrollView(horizontal)` + `Row`
- 改为 **`ConstrainedBox` 限定最大高度（如 240px）+ `SingleChildScrollView(vertical)` + `Column` 或 `Wrap`**
- 每个时间线卡片（`_compactTimelineCard`）保持 160px 宽，以 `Wrap` 排列，超出容器高度时垂直滚动
- 视觉上保留箭头指示流向，但箭头只出现在同一行内相邻项之间
- 添加微弱的滚动指示器（Scrollbar），提示可滚动

### UI 伪代码
```
┌──────────────────────────────┐
│ 时间线                   ← scrollbar
│ ┌─────┐ → ┌─────┐ → ┌─────┐│
│ │卡1  │   │卡2  │   │卡3  ││
│ └─────┘   └─────┘   └─────┘│
│ ┌─────┐ → ┌─────┐         │
│ │卡4  │   │卡5  │         │
│ └─────┘   └─────┘         │
└──────────────────────────────┘
```

### 状态管理
不需要额外状态，Container 自带 scrollable。

## 计划表模块（`_buildPlanTable`）改造

### 现状
```dart
Column(children: [header, ...rows])
```
所有行一次性渲染，行数多时卡片超高。

### 目标
每页固定显示 5 行，超过分页。

### 设计
- 新增状态变量 `_planTablePageIndex`（Map 或局部变量）
- 每页显示 5 行
- 表头固定
- 底部添加分页控件：`<< < Page 1/3 > >>` 或 `Row([prev, pageInfo, next])`
- 切换页码时 `setState` 更新

### UI 伪代码
```
┌──────────────────────────────────┐
│ 计划表                           │
│ ┌──────┬──────┬──────────────┐   │
│ │ 时间  │ 主题  │ 训练内容     │   │
│ ├──────┼──────┼──────────────┤   │
│ │ ...  │ ...  │ ...          │   │
│ │ ...  │ ...  │ ...          │   │
│ │ ...  │ ...  │ ...          │   │
│ │ ...  │ ...  │ ...          │   │
│ │ ...  │ ...  │ ...          │   │
│ └──────┴──────┴──────────────┘   │
│      ← 1/3 →                     │
└──────────────────────────────────┘
```

### 分页状态
由于 `_buildPlanTable` 在 `_buildPlanCard` 中，且每个计划卡片独立，分页状态需要按消息索引存储。

设计方案：
- 在 `_messages` 列表中每个消息的 map 中存储 `planTablePage` 字段
- 或使用 `_messages[index]['planTablePage']` 记录当前页
- 这样每个计划卡片的分页状态独立互不影响

## 边界情况
- **0 行**：不渲染计划表和时间线
- **≤5 行**：不分页，不显示分页控件
- **时间线项 ≤3**：不启用滚动，自然展示
- **高度自适应**：不同屏幕尺寸下仍保持合理的 maxHeight

## 不变的内容
- 消息气泡样式、颜色、交互行为不变
- 计划卡片其他模块（WBS 思维导图、原始计划展开、建议按钮）不变
- 消息列表滚动行为不变

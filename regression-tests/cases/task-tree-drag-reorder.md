# 回归测试：task-tree-drag-reorder

## 测试环境
- App: SmartAssistant (Flutter)
- 测试方式: 手工验证

## 前置条件
- 数据库中有至少 3 个任务，包括父子关系
  - 任务 A (root, status=0)
  - 任务 A1 (parent=A, status=0)  
  - 任务 B (root, status=0)
  - 任务 C (root, status=2, 已完成)

## 测试用例

### TC1: 树形结构展示
1. 打开任务列表页
2. **预期**: 任务 A、B 平级展示，A1 在 A 下方缩进显示
3. **预期**: A 左侧显示展开箭头 `▼`
4. **预期**: B 左侧无展开箭头（无子任务）
5. **预期**: 已完成任务 C 在折叠的"已完成"区域内，不参与树形

### TC2: 展开/折叠
1. 点击任务 A 的展开箭头
2. **预期**: A1 隐藏，箭头变为 `▶`
3. 再次点击箭头
4. **预期**: A1 重新显示，箭头变为 `▼`

### TC3: 拖拽子任务到其他父任务
1. 长按任务 A1 的拖拽手柄（六点图标）
2. 拖动到任务 B 上，任务 B 应高亮
3. 释放
4. **预期**: A1 移动到 B 下方，缩进显示为 B 的子任务

### TC4: 拖拽子任务到根级别
1. 确保 A1 是某个任务的子任务
2. 长按 A1 的拖拽手柄
3. 拖动到顶部的"拖到此处移为根任务"区域
4. **预期**: A1 变为根任务（平级），不再缩进

### TC5: 不能拖到自己身上
1. 长按任务 A 的拖拽手柄
2. 拖动到任务 A 自身上
3. **预期**: 无操作，A 不高亮

### TC6: 筛选下树形正确
1. 切换到某个项目筛选
2. **预期**: 树形仅在当前项目任务内构建
3. 切换到"今天"筛选
4. **预期**: 树形在筛选结果内正确

### TC7: 创建子任务
1. 点击 + 按钮
2. 在创建表单中选择父任务（如 A）
3. 提交
4. **预期**: 新任务出现在 A 下方缩进

## 回归通过条件
- 以上所有用例通过

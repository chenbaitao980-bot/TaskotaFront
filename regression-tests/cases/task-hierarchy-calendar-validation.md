# 回归测试：task-hierarchy-calendar-validation

## 测试场景

### 场景 1：父任务不展示在日历
1. 创建一个新任务，在创建页面开启"标记为父任务"开关
2. 保存后切换到日历页面
3. 验证：该任务不出现在周视图/月视图的时间线中

### 场景 2：子任务展示父任务名
1. 创建一个父任务
2. 在父任务下创建一个子任务（有日期时间）
3. 切换到日历页面
4. 验证：子任务的时间块上显示 "📁 父任务名" 标签

### 场景 3：时间冲突弹窗拦截
1. 创建一个日程 09:00-10:00
2. 再创建一个子任务 09:30-10:30
3. 验证：弹出时间冲突提示对话框

### 场景 4：折叠分组可展开收起
1. 创建一个父任务
2. 在父任务下创建 2-3 个子任务
3. 切换到日历周视图
4. 验证：多日任务栏下方显示父任务分组，可点击展开/收起

### 场景 5：父子属性自动切换
1. 创建一个普通任务（不带子任务）
2. 在该任务下添加一个子任务
3. 验证：该任务自动变成父任务，从日历上消失
4. 删除所有子任务
5. 验证：该任务恢复为普通任务，重新出现在日历上

### 场景 6：首页过滤父任务
1. 在首页查看待办计数
2. 创建一个父任务（有子任务）
3. 验证：首页待办计数不包含该父任务
4. 在今日日程区域查看
5. 验证：子任务按父任务分组展示

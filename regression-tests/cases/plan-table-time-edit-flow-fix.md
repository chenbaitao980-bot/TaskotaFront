# 回归用例：plan-table-time-edit-flow-fix

## 背景
AI 助手生成一周训练计划后，计划表、流程图和一键分配任务必须使用同一份可编辑计划数据，避免展示和任务分配不一致。

## 覆盖范围
- 计划表展示开始时间和结束时间，精确到分钟。
- 计划表中的主题、训练内容、开始时间、结束时间可以直接修改。
- 流程图或思维导图展示全部计划节点，不再只显示到周日或固定 7 个节点。
- 一键分配任务时，任务标题使用每行训练主题或训练内容，不把表头“主题”错误带入标题。

## 手工验证步骤
1. 在 AI 助手中输入“给我生成一周足球训练计划，每天 30 分钟，开始时间 09:00”。
2. 确认返回计划表包含“时间、主题、训练内容”，每行时间显示到分钟，例如 `05-25 09:00 - 05-25 09:30`。
3. 修改任意一行的主题、训练内容、开始时间和结束时间。
4. 打开流程图或思维导图，确认修改后的全部计划节点都展示出来。
5. 点击一键分配任务，确认预览标题不出现“主题”，并且使用用户编辑后的主题或训练内容。

## 通过标准
- 静态检查通过：`flutter analyze --no-fatal-infos`。
- 自动测试通过：`flutter test`。
- 用户验证四个问题均已修复。

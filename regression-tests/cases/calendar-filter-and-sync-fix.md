# 回归测试：日历项目筛选与拖拽同步修复

## 测试范围
- 日历项目筛选可切回"全部项目"
- 日历拖拽时间后切换到任务页时间同步
- 时间线任务块拖拽热区调整大小
- 跨日任务条拖拽移动/调整

## 测试步骤

### 1. 项目筛选切回全部项目
1. 打开日历页面
2. 点击项目筛选图标，选择任意一个项目
3. 确认日历只显示该项目下的任务
4. 再次点击项目筛选图标，选择"全部项目"
5. 确认日历恢复显示所有项目下的任务

### 2. 日历拖拽后任务页时间同步
1. 在日历时间线上找到有持续时间的任务块
2. 拖拽该任务块到其他时间位置
3. 切换到"任务"页面（底部导航第2个tab）
4. 确认该任务的起止时间已更新为拖拽后的时间

### 3. 时间线任务块拖拽热区
1. 在日历周视图中找到单日任务块
2. 将鼠标悬停在任务块顶部区域（约20px高度）
3. 确认出现调整手柄（白色细条）
4. 拖拽手柄调整开始时间，确认任务块高度变化有实时预览
5. 释放后确认开始时间已更新
6. 重复底部手柄调整结束时间

### 4. 跨日任务条拖拽调整
1. 确保存在跨日任务（startDate与dueDate不在同一天）
2. 在跨日任务条区域找到该任务
3. 拖拽整个任务条到新的起始日位置
4. 确认任务起止时间同步移动
5. 将鼠标悬停在任务条左侧边缘（约18px区域）
6. 拖拽左侧手柄调整开始日期
7. 确认开始日期更新且结束日期不变
8. 拖拽右侧手柄调整结束日期
9. 确认结束日期更新且开始日期不变

## 预期结果
- 筛选项目后可正常切回"全部项目"
- 日历拖拽时间后任务页显示同步更新时间
- 时间线任务块顶部和底部有20px可拖拽热区，拖拽时有实时预览
- 跨日任务条可整体拖拽移动，左右侧有手柄可独立调整开始/结束日期
- flutter analyze 无新增警告
